# Spec: 基本的なファイル操作の実装

## 概要
このトラックでは、Electronメモアプリケーションにおける基本的なファイル操作機能（ファイルの読み込み、新規作成、保存）を実装します。特に、Electronのメインプロセスとレンダラープロセス間の通信に`ipcMain.on`を活用し、セキュアかつ効率的なバックエンド処理を実現します。

## ゴール
- ユーザーが既存のテキストファイルをアプリケーションで開けるようにする。
- ユーザーが新規テキストファイルを作成し、内容を編集できるようにする。
- ユーザーが編集した内容をファイルとして保存できるようにする。

## 詳細
- **ファイルの読み込み:**
    - レンダラープロセスから開くファイルダイアログをトリガーし、ユーザーがファイルを選択できるようにする。
    - 選択されたファイルのパスを`ipcRenderer`を通じてメインプロセスに送信する。
    - メインプロセスは指定されたファイルを非同期で読み込み、その内容を`ipcRenderer`を通じてレンダラープロセスに返送する。
    - レンダラープロセスは受け取った内容をエディタに表示する。
- **新規ファイルの作成:**
    - レンダラープロセスからの要求に応じて、メインプロセスは新規の空ファイルを作成する準備をする。
    - 保存ダイアログを表示し、ユーザーが保存場所とファイル名を指定できるようにする。
- **ファイルの保存:**
    - レンダラープロセスから、編集中のテキスト内容と保存先のファイルパス（または新規保存のための情報）をメインプロセスに送信する。
    - メインプロセスは受け取った内容を指定されたファイルに非同期で書き込む。
    - 成功または失敗のフィードバックをレンダラープロセスに返送する。

## 技術的考慮事項
- `ipcMain.on` と `ipcRenderer.invoke`/`ipcRenderer.handle` を利用したセキュアなプロセス間通信。
- 非同期ファイル操作にはNode.jsの`fs`モジュールを使用。
- レンダラープロセス側ではReactのstate管理を活用し、エディタの内容を保持。
